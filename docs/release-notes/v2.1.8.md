# Codex App Transfer v2.1.8

## 中文

### 转发状态显示与启停交互优化

修复转发状态显示不准确的问题：Dashboard 中“已停止”状态不再错误显示为绿色，Proxy 页面停止态也会同步变更为灰色视觉反馈，避免用户误判服务仍在运行。

同时将 Proxy 页面原本分离的“启动转发 / 停止转发”两个按钮合并为单一 toggle 按钮，运行中显示“停止转发”，停止时显示“启动转发”，交互更直观。

主要改动:

- Dashboard 转发状态卡片改为根据真实 `proxyRunning` 动态切换图标颜色、状态文字与活动徽标。
- Proxy 页面启停按钮合并为单一 toggle 按钮，状态、图标和按钮样式随运行状态实时切换。
- 停止转发时，Proxy 页面 pulse-dot 与状态文字统一变灰，强化“已停止”反馈。
- `stopProxy()` 前端调用补充 `/api/status` 二次确认，避免仅依赖本地硬编码状态。
- 后端在启停成功时写入生命周期日志，日志面板可直接看到“forwarding started :PORT” 与 “forwarding stopped”。

### 验证

- `cargo check --manifest-path Cargo.toml`

## English

### Forwarding status UI and start/stop interaction improvements

Fixes inaccurate forwarding status display: the Dashboard no longer shows the stopped state in green, and the Proxy page now switches to a muted stopped-state visual so users can clearly see when forwarding is not running.

The Proxy page also merges the separate “Start Forwarding / Stop Forwarding” buttons into a single toggle button. It shows “Stop Forwarding” while running and “Start Forwarding” while stopped, making the interaction more intuitive.

Changes:

- Dashboard forwarding status card now updates icon color, status text, and activity badge from real `proxyRunning` state.
- Proxy page start/stop controls are merged into a single toggle button with dynamic label, icon, and style.
- Stopped state now mutes the pulse-dot and status text on the Proxy page.
- Frontend `stopProxy()` now re-checks `/api/status` instead of relying on a hardcoded local result.
- Backend emits lifecycle log entries on successful start/stop so the log panel shows “forwarding started :PORT” and “forwarding stopped”.
